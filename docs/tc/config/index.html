              # 偽靜配置

              後臺 => 系統 => URL地址配置 => 路由偽靜態設置

              • 路由狀態：開啟
              • 偽靜態狀態：開啟

              # rewrite說明

              • 1，確認空間支持rewrite組件。
              • 2，按照 偽靜態rewrite 目錄下的說明文檔操作。
              • 3，後臺設置瀏覽模式為 rewrite偽靜態 。

              如果使用rewrite偽靜態模式，請註意把配置文件復製到網站根目錄。如果靜態文件後綴配置的不是html，則請把rewrite配置文件中的.html替換為對應的後綴如.html

              • iis6.x 下使用 httpd.ini
              • iis7.x 下使用web.config

              # apache下使用 .htaccess

              (編輯.htaccess文件，把 RewriteBase /maccms10 修改為你蘋果CMS所在目錄)

              # nginx 下使用 maccms.conf

              (使用vps或者服務器的可以在你的主機的conf裏 用 include xxxxx.conf 也就是包含下偽靜態規則文件 如果用的是虛擬主機版的nginx 就找你的主機商給你添加規則就行，你把規則發給他。)

              # 郵件發送

              蘋果CMS v10在使用的過程中，我們可能需要通過郵件驗證用戶是否為機器人，通過郵件驗證身份是一個不錯的選擇，大部分國內服務器是禁用了服務器發件功能的，直接發送郵件是無法實現的。我們只能通過配置smtp郵件服務器來完成郵件的發送。

              蘋果CMS 10在開發的過程中已經做好了相應的功能組件，我們只需要填寫我們的smtp發件服務器信息即可向用戶推送郵件。

              擴展庫支持

              蘋果CMS v10使用smtp發送郵件需要開啟OpenSSL擴展庫支持，否則會發送失敗。這裏小夥伴們不需要擔心，因為現在的服務器openssl擴展庫是必備的，已經默認內置了。

              我們填入我們所使用的smtp發件服務器信息保存即可，這裏提醒使用QQ郵箱、126郵箱、163郵箱等常用郵箱的小夥伴一點，密碼那一欄填寫的並不是真正的郵箱密碼，而是你在郵箱設置中獲得的郵箱授權碼。

              1，我們依次打開蘋果CMSv10後臺設置，依次在左側導航欄找到 郵件發送配置 並打開，我們會看見如同所示的界面。

              2.我們一QQ郵箱作為發送服務 配置 PhpMailer

              • 服務器：smtp.qq.com
              • 端口：587
              • 傳輸協議：TSL
              • 帳號：你的qq郵箱
              • 密碼: 你的郵箱登錄密碼

              2，測試

              填寫完成後我們可以使用發送郵件測試功能，來測試配置是否正確，我們在這裏做了一個簡單地小測試，結果非常完美。

              # 短信發送

              短信發送功能類似於郵箱發送能，也是用於給用戶發送驗證碼短信，手機註冊會員等功能。目前蘋果CMS 後臺已經集成了阿裏雲短信和騰訊雲短信對接服務器，只需申請相關key即可使用！阿裏雲購買流程沒有什麽好說明的，下一步下一步然後更具自己的購買短信量即可，國內這兩家短信發送服務都是需要網站實名備案才可以使用哦。

              # 申請阿裏雲短信包

              阿裏雲短信申請地址：https://www.aliyun.com/product/sms (opens new window)

              夠買成功後需要配置一下相關的簽名和發送模板。

              短信服務的簽名都需要經過審核，審核通過後才可以使用該簽名。本文將為您介紹短信簽名的規範說明，幫助您快速通過簽名審核。

              添加簽名： https://dysms.console.aliyun.com/dysms.htm#/domestic/text/sign (opens new window)

              簽名就是一什麽企業名稱在發送短信，告訴用戶是誰發的短信。例如：大圖模板，那麽用戶收到的內容就是：

              【大圖模板】您正在申請手機註冊，驗證碼為 12456 ，5分鐘內有效。
              1

              其中內容格式由模板決定。

              等審核通過後就可以添加模板了

              添加模板：https://dysms.console.aliyun.com/dysms.htm#/domestic/text/template (opens new window)

              下圖示例為添加手機註冊模板：

              阿裏雲短信模板例子

              • 尊敬的用戶，您的註冊會員驗證碼為：${code}，請勿泄漏於他人！
              • 驗證碼為：${code}，您正在綁定手機，若非本人操作，請勿泄露。
              • 驗證碼為：${code}，您正在進行密碼重置操作，如非本人操作，請忽略本短信！

              獲取keyid 和 KeySecret

              以上工作都準備好了以後，到阿裏雲的 RAM 訪問控製：https://ram.console.aliyun.com/users (opens new window)創建一個用戶

              • 創建

              • 添加 AliyunDysmsFullAccess 和 AliyunDysmsReadOnlyAccess 兩條策略

              • 點擊創建 AccessKey , 裏面的 AccessKey ID 和 AccessKey Secret 就是我們要填寫在蘋果cms後臺的

              # 視頻試看配置

              蘋果cmsV10如何配置遊客試看功能必備條件：

              須知條件

              • 1.程序：一些源碼交易網站上有肯能會有整站的程序下載，一般這些源碼更新不及時，甚至被人動過手腳，所以建議官方最新版。
              • 2.模板：並不是說有的模板都支持試看，尤其是網上一下來歷不明的模板，各種bug殘缺不全，功能測試建議用官方自帶的模板，官方模板基本上都是全功能的。

              # 遊客開啟試看1分鐘：

              • 第一步：在後臺的 -> 系統 -> 網站參數配置 ->試看時長 -> 設置為 1

              • 第二步：在後臺 的 用戶 -> 會員組 -> 找到遊客用戶組 -> 點擊編輯按鈕

              可根據分類單獨配置試看權限，我這裏是全站數據都對遊客開啟試看，所以把所有分類的播放頁勾去掉了，剩下的全打勾，點擊保存設置，這樣試看功能就配置成功了

              列表頁、內容頁、播放頁、下載頁4個權限，控製是否可以進入頁面，沒權限會直接返回提示信息。試看權限：如果沒有訪問播放頁的權限、或者有權限但是需要積分購買的數據，開啟了試看權限也是可以進入頁面的。開啟試看功能的話，播放窗口將采用iframe動態頁面方式載入，如果你需要對vip用戶開啟播放全片權限，只需要對vip用戶組的權限全部打鉤就ok了。

              提示

              ，默認的方式都是以進入頁面就開始計時，不管你有沒有播放視頻，時間到了就會銷毀播放器。更精確的試看功能是需要配合播放器的進度條進行監聽完成，市面上黑多H5網頁播放器都支持實時監聽進度時間。

              # 三級分銷配置

              三級分銷功能不被很多站長看中，可能是有的模板沒開發好這部分的功能吧。其實這個裂變推廣非常實用，用得好可以給網站帶來可觀的流量。很多人不了解蘋果cmsv10中的三級分銷實現方法和計算比例，也有可能是三級分銷功能不被看好的原因。

              # 如何有效關聯？

              只要您的好友通過你的邀請鏈接訪問後並且註冊成會員，你的好友既是你的一級分銷，當您的好友有積分消費時，你就可以獲得一級分銷提成。您的好友如果在邀請他的好友，他的好友註冊成會員並且消費了積分 你還可以獲得二級積分獎勵，他的好友如果在邀請其他人註冊成會員消費，你還可以獲得三級積分獎勵，有效關聯最高達到三層，這就是三級分銷分銷關聯機製。

              • A 邀請了 B，B是A的一級分銷
              • B 邀請了 C，C是B的一級分銷、是A的二級分銷
              • C 邀請了 D，D是C的一級分銷、是B的二級分銷、是A的三級分銷

              綁定關系永久有效

              # 積分獎勵比例計算公式

              • 一級設置比二級大，二級設置比三級大。
              • 一級比例：5 二級比例：3 三級比例：2

              計算公式源碼

              # $fee_points 消費的積分
# $GLOBALS['config']['user']['reward_ratio'] 後臺配置比例
$points = floor($fee_points / 100 * $GLOBALS['config']['user']['reward_ratio']);

              定時任務目前內置了 采集資源、生成靜態 2個執行操作 強製執行參數enforce=1

              # 1，采集資源

              • 執行文件：選擇 采集資源庫collect

              • 附加參數：從自定義資源列表中獲取（在采集今日，采集本周，采集全部 右鍵復製鏈接）截取參數部分即可。

              • 例如：

              • 任務名稱：cj_day

              • 任務描述：采集當天數據

              • 附加參數： ac=cjall&h=24&xt=1&ct=&cjflag=b9c546ba925d22ed654927b44638df34&cjurl=http://cj.tv6.com/mox/inc/youku.php

              # 2，生成靜態

              • 執行文件：選擇生成make

              • 附加參數：

              • 例如：

              • 生成首頁 ac=index

              • 生成地圖頁 ac=map

              • 生成rss ac=rss

              • 生成百度sitemap ac=rss&ac2=baidu

              • 生成谷歌sitemap ac=rss&ac2=google

              • 生成專題首頁 ac=topic_index

              • 生成專題詳情頁 ac=topic_info&topic=1,2,3,4

              • 生成視頻分類頁 ac=type&tab=vod&vodtype=1,2

              • 生成當日有更新數據的視頻分類 ac=type&tab=vod&ac2=day

              • 生成文章分類頁 ac=type&tab=art&arttype=3,4

              • 生成當日有更新數據的文章分類 ac=type&tab=art&ac2=day

              • 生成自定義頁面 ac=label&label=rand.html

              • 生成視頻詳情頁 ac=info&tab=vod&ids=1,2,3

              • 生成未生成視頻詳情頁 ac=info&tab=vod&ac2=nomake

              • 生成文章詳情頁 ac=info&tab=art&ids=1,2,3

              • 生成未生成文章詳情頁 ac=info&tab=art&ac2=nomake

              # 3，采集規則

              • 執行文件：選擇 采集規則cj
              • 參數id=1，參數就是當前采集自定義采集規則的編號。
              • 為了不影響服務器性能，目前僅采集第一頁。

              • 無需參數
